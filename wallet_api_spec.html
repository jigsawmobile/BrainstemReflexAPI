Endpoints

1. Register Wallet

POST/register

Request

{
  "public_address": "0xYourEthereumAddress"
}

Response — Success

{
  "status": "success",
  "nonce": 12345
}

Response — Already exists

{
  "status": "failed",
  "message": "already exists"
}

2. Get Nonce

GET/get-nonce?public_address=0x...

Response — Exists

{
  "status": "exists",
  "nonce": 12345
}

Response — Not found

{
  "status": "not found"
}

3. Login

POST/login

Request

{
  "public_address": "0x...",
  "signature": "0xSignature",
  "source": "brainstem"
}

Response

{
  "status": "success",
  "access_token": "token",
  "refresh_token": "refresh"
}

Error

{
  "message": "Signature invalid"
}

4. Refresh Token

POST/refresh

Headers

Authorization: Bearer <refresh_token>

Response

{
  "access_token": "new_token"
}

5. Upload Record

POST/api/v1/upload-record

Headers

Authorization: Bearer <access_token>
Content-Type: multipart/form-data

Body

file: <binary>

Success

{
  "id": "file_id",
  "message": "File successfully uploaded"
}

Duplicate file

{
  "status": "failed",
  "messages": ["The uploaded file already exists in the dataset."]
}

Token expired

{
  "msg": "Token has expired"
}

Error Handling

Token Expiration

{
  "msg": "Token has expired"
}

Generic Failure

{
  "status": "failed",
  "messages": ["..."]
}

Notes

  • Ethereum address uses EIP-55 checksum format.
  • Nonce must be signed as a plain string.
  • Rewards are returned in Wei and converted client-side.
  • /claim has inconsistent response formats.
  • Protected endpoints require token refresh when expired.
